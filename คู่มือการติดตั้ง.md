# คู่มือติดตั้งและใช้งาน — คลังภาพกิจกรรมโรงเรียน

เว็บรวมลิงก์อัลบั้ม Google Drive พร้อมภาพตัวอย่าง มุมมองปฏิทิน และแผงผู้ดูแล
สถาปัตยกรรม: **GitHub Pages (หน้าเว็บ) + Google Apps Script (API) + Google Sheets (ฐานข้อมูล) + Google Drive (คลังรูป)**

ไฟล์ทั้งหมดในชุดนี้:

| ไฟล์ | หน้าที่ | ติดตั้งที่ |
|---|---|---|
| `Code.gs` | JSON API อ่าน/เขียนข้อมูล + สแกนรูปจาก Drive | Google Apps Script |
| `index.html` | หน้ารวมอัลบั้มสำหรับผู้ใช้ (กริด + ปฏิทิน) | GitHub Pages |
| `admin.html` | แผงผู้ดูแลสำหรับครู | GitHub Pages |
| `style.css` | ธีมสว่าง สะอาด ทางการ (ใช้ร่วมกัน) | GitHub Pages |
| `app.js` | logic หน้าผู้ใช้ | GitHub Pages |
| `admin.js` | logic แผงผู้ดูแล | GitHub Pages |

---

## ส่วนที่ 1 — ติดตั้งฝั่ง Google (ทำครั้งเดียว ~10 นาที)

### 1.1 สร้าง Google Sheet + Apps Script

1. สร้าง Google Sheet ใหม่ ตั้งชื่ออะไรก็ได้ เช่น `GalleryHub Database`
2. เมนู **ส่วนขยาย (Extensions) → Apps Script**
3. ลบโค้ดเดิมทั้งหมด แล้ววางโค้ดจากไฟล์ `Code.gs`
4. กดบันทึก (ไอคอนแผ่นดิสก์)

### 1.2 รัน setup() หนึ่งครั้ง

1. ที่แถบด้านบน เลือกฟังก์ชัน `setup` แล้วกด **เรียกใช้ (Run)**
2. ครั้งแรกจะมีหน้าต่างขออนุญาต → กด **Review permissions** → เลือกบัญชี →
   ถ้าขึ้นหน้า "Google hasn't verified this app" ให้กด **Advanced → Go to ... (unsafe)** → **Allow**
   (เป็นสคริปต์ของเราเอง ปลอดภัย)
3. ระบบจะสร้างชีต `Albums` พร้อมหัวตาราง และตั้งค่า username/password เริ่มต้นให้

> **เปลี่ยนรหัสผ่าน:** ไปที่ **Project Settings (ไอคอนเฟือง) → Script Properties**
> แก้ค่า `ADMIN_USER` และ `ADMIN_PASS` ได้ตามต้องการ
> ส่วน `ADMIN_TOKEN` คือกุญแจลับของระบบ **ห้ามเผยแพร่** (ไม่ต้องจำ ระบบใช้เองอัตโนมัติ)

### 1.3 Deploy เป็น Web App

1. กดปุ่ม **Deploy → New deployment**
2. กดไอคอนเฟือง เลือกประเภท **Web app**
3. ตั้งค่าดังนี้ (สำคัญมาก ผิดแล้วเว็บเชื่อมต่อไม่ได้):
   - **Execute as:** `Me` (บัญชีของเรา)
   - **Who has access:** `Anyone`
4. กด **Deploy** แล้ว **คัดลอก Web app URL** เก็บไว้
   (หน้าตาแบบ `https://script.google.com/macros/s/AKfycb.../exec`)

> **เมื่อแก้โค้ด Code.gs ภายหลัง:** ต้อง Deploy ใหม่แบบ
> **Deploy → Manage deployments → ✏️ Edit → Version: New version → Deploy**
> ถ้าสร้าง New deployment ใหม่ URL จะเปลี่ยน แล้วต้องไปแก้ในเว็บด้วย

## ส่วนที่ 2 — ติดตั้งฝั่ง GitHub Pages (~10 นาที)

### 2.1 ใส่ URL ของ API ลงในโค้ด

เปิดไฟล์ `app.js` และ `admin.js` แก้บรรทัดบนสุด:

```javascript
const API_URL = 'วาง URL ของ GAS Web App ที่นี่';
```

เปลี่ยนเป็น URL ที่คัดลอกไว้จากข้อ 1.3 (ต้องเหมือนกันทั้ง 2 ไฟล์)

### 2.2 สร้าง repository และเปิด Pages

1. เข้า [github.com](https://github.com) → **New repository** → ตั้งชื่อ เช่น `school-gallery` → Public → Create
2. กด **uploading an existing file** แล้วลากไฟล์ทั้ง 5 ไฟล์นี้ขึ้นไป:
   `index.html`, `admin.html`, `style.css`, `app.js`, `admin.js`
3. กด **Commit changes**
4. ไปที่ **Settings → Pages** → Source เลือก **Deploy from a branch** →
   Branch เลือก `main` / `(root)` → **Save**
5. รอ 1–2 นาที เว็บจะออนไลน์ที่
   `https://ชื่อผู้ใช้.github.io/school-gallery/`

### 2.3 ทดสอบระบบ

1. เปิดเว็บ → ควรเห็นหน้าคลังภาพ (ยังไม่มีอัลบั้ม ขึ้นข้อความว่าง เป็นเรื่องปกติ)
2. กด **ผู้ดูแล** → เข้าสู่ระบบด้วย username/password จาก Script Properties
3. ลองเพิ่มอัลบั้มแรก (ดูวิธีในส่วนที่ 3)

## ส่วนที่ 3 — วิธีใช้งานสำหรับครูผู้ดูแล

### เพิ่มอัลบั้มใหม่

1. อัปโหลดรูปกิจกรรมเข้าโฟลเดอร์ใน Google Drive ตามปกติ
2. คัดลอก **ลิงก์โฟลเดอร์** (คลิกขวาโฟลเดอร์ → แชร์ → คัดลอกลิงก์)
3. เข้าเว็บ → **ผู้ดูแล** → เข้าสู่ระบบ
4. วางลิงก์ในช่อง "ลิงก์โฟลเดอร์ Google Drive"
5. กรอกชื่อกิจกรรม, หมวดหมู่, **วันที่จัดกิจกรรม** (สำคัญ — ใช้แสดงบนปฏิทิน), ปีการศึกษา
6. กด **เพิ่มอัลบั้ม** → ระบบจะสแกนโฟลเดอร์ นับจำนวนรูป และดึงรูปปกให้อัตโนมัติ
   พร้อมตั้งค่าแชร์โฟลเดอร์เป็น "ทุกคนที่มีลิงก์" ให้เอง (ถ้าบัญชีเป็นเจ้าของโฟลเดอร์)

> **สำคัญเรื่องการแชร์:** รูปตัวอย่างจะแสดงได้ต่อเมื่อโฟลเดอร์แชร์แบบ
> **"ทุกคนที่มีลิงก์ — ผู้มีสิทธิ์ดู (Anyone with the link — Viewer)"**
> ถ้าเพิ่มอัลบั้มแล้วรูปปกไม่ขึ้น ให้ไปตั้งค่าแชร์โฟลเดอร์เองใน Drive

### จัดการอัลบั้มเดิม

- **สแกนใหม่** — ใช้เมื่อเพิ่มรูปเข้าโฟลเดอร์ทีหลัง เพื่ออัปเดตจำนวนรูปและรูปปก
- **ซ่อน / แสดง** — ซ่อนอัลบั้มจากหน้าเว็บชั่วคราวโดยไม่ต้องลบ
- **ลบ** — ลบเฉพาะรายการบนเว็บ **ไฟล์รูปใน Drive ไม่ถูกลบ**
- ตัวเลข **วิว** บอกจำนวนครั้งที่มีคนกดเปิดอัลบั้มนั้น

## ส่วนที่ 4 — วิธีใช้งานสำหรับนักเรียน/ผู้ปกครอง

- **มุมมองอัลบั้ม** — เลื่อนดูการ์ด กดการ์ดเพื่อเปิดดูรูปทั้งหมดใน Google Drive
- **ค้นหา** — พิมพ์ชื่อกิจกรรมในช่องค้นหา
- **กรอง** — กดป้ายหมวดหมู่ หรือเลือกปีการศึกษา
- **มุมมองปฏิทิน** — กดปุ่ม "ปฏิทิน" มุมขวา วันที่มีจุดสีน้ำเงินคือวันที่มีอัลบั้ม
  กดวันนั้นเพื่อดูรายการอัลบั้มด้านล่าง แล้วกดเปิดเข้า Drive ได้เลย
  ใช้ปุ่มลูกศรเลื่อนดูเดือนอื่น ปุ่ม "วันนี้" กลับมาเดือนปัจจุบัน
  (วันนี้จะมีวงแหวนสีทองกำกับ)

## ส่วนที่ 5 — ระบบค้นหาใบหน้า (ค้นหารูปตัวเอง)

ผู้ใช้ถ่ายเซลฟี่ แล้วระบบจะหารูปที่มีใบหน้าของเขาจากทุกอัลบั้ม
**รูปเซลฟี่ประมวลผลในเครื่องของผู้ใช้เท่านั้น ไม่ถูกอัปโหลดขึ้นระบบ**
สิ่งที่เก็บในชีต `FaceIndex` คือชุดตัวเลข 128 ตัวต่อใบหน้า (descriptor) ไม่ใช่รูปภาพ

### 5.1 อัปเดตระบบเดิม (กรณีติดตั้งเวอร์ชันก่อนหน้าไปแล้ว)

1. วางโค้ด `Code.gs` เวอร์ชันใหม่ทับของเดิมใน Apps Script
2. **รัน `setup()` อีกครั้ง** — เพื่อสร้างชีต `FaceIndex` (ข้อมูลอัลบั้มเดิมไม่หาย)
3. Deploy แบบ **Manage deployments → ✏️ Edit → Version: New version → Deploy**
   (ห้ามสร้าง New deployment — URL จะเปลี่ยน)
4. อัปโหลดไฟล์เว็บทั้ง 5 ไฟล์เวอร์ชันใหม่ทับของเดิมบน GitHub
   (อย่าลืมแก้ `API_URL` ใน `app.js` และ `admin.js` ก่อนอัปโหลด)

ถ้าติดตั้งใหม่ทั้งหมด ทำตามส่วนที่ 1–2 ตามปกติ ระบบใบหน้าจะพร้อมใช้เลย

### 5.2 ขั้นตอนของครูผู้ดูแล: สร้าง "ดัชนีใบหน้า"

ระบบค้นหาจะทำงานได้ก็ต่อเมื่ออัลบั้มนั้นมีดัชนีแล้ว:

1. เข้าแผงผู้ดูแล → ที่แถวอัลบั้ม กดปุ่ม **ดัชนีใบหน้า**
2. ระบบจะบอกจำนวนรูปที่ต้องสแกนและเวลาโดยประมาณ → กดตกลง
3. **เปิดหน้านี้ค้างไว้จนสแกนเสร็จ** (เบราว์เซอร์ของครูเป็นคนสแกน
   ปุ่มจะแสดงความคืบหน้า เช่น "สแกน 45/248 (พบ 120 หน้า)")
4. ถ้าปิดหน้ากลางคัน ไม่เสียหาย — กดปุ่มเดิมอีกครั้ง ระบบจะ**สแกนต่อจากที่ค้าง**
   เฉพาะรูปที่ยังไม่มีในดัชนี
5. เมื่อเพิ่มรูปใหม่เข้าโฟลเดอร์ภายหลัง กด **ดัชนีใบหน้า** ซ้ำ
   ระบบจะสแกนเฉพาะรูปใหม่ให้เอง

ข้อแนะนำ: สแกนบนคอมพิวเตอร์จะเร็วกว่ามือถือมาก และอัลบั้มละประมาณ 1–2 วินาทีต่อรูป

### 5.3 ขั้นตอนของผู้ใช้: ค้นหารูปตัวเอง

1. เปิดหน้าเว็บ → กดแถบ **ค้นหารูปตัวเอง** ใต้ช่องค้นหา
2. รอโหลดโมเดล AI (ครั้งแรกอาจ 10–20 วินาที ครั้งต่อไปเร็วขึ้น)
3. อนุญาตให้เว็บใช้กล้อง → จัดใบหน้ากลางกรอบ → กด **ถ่ายและค้นหา**
4. ระบบแสดงรูปทั้งหมดที่พบ จัดกลุ่มตามอัลบั้ม กดรูปเพื่อเปิดดูเต็มใน Drive
5. เคล็ดลับ: ถ้าเลือกตัวกรองหมวดหมู่หรือปีการศึกษาไว้ก่อนกดค้นหา
   ระบบจะค้นเฉพาะอัลบั้มตามตัวกรอง ทำให้เร็วขึ้น

> **หมายเหตุการใช้กล้อง:** เบราว์เซอร์อนุญาตกล้องเฉพาะเว็บ `https://` ซึ่ง
> GitHub Pages เป็น https อยู่แล้ว ใช้ได้ทั้งมือถือและคอมพิวเตอร์

## ส่วนที่ 6 — นำเข้าหลายอัลบั้มจากโฟลเดอร์ใหญ่ (Bulk Import)

สำหรับย้ายรูปเก่าทั้งปีเข้าระบบโดยไม่ต้องเพิ่มทีละอัลบั้ม:
เตรียมโฟลเดอร์ใหญ่ 1 โฟลเดอร์ (เช่น "รูปกิจกรรม ปีการศึกษา 2569")
ข้างในเป็นโฟลเดอร์ย่อย **1 โฟลเดอร์ = 1 กิจกรรม**

### วิธีใช้

1. เข้าแผงผู้ดูแล → ส่วน **นำเข้าหลายอัลบั้มจากโฟลเดอร์ใหญ่**
2. วางลิงก์โฟลเดอร์ใหญ่ → กด **สแกนโฟลเดอร์ย่อย**
3. ระบบแสดงตารางรายการทั้งหมด พร้อมเดาข้อมูลให้อัตโนมัติ:
   - **ชื่อกิจกรรม** = ชื่อโฟลเดอร์ (ตัดวันที่/วงเล็บหมวดออกให้)
   - **วันที่จัด** = อ่านจากชื่อโฟลเดอร์ ถ้าไม่มีใช้วันที่สร้างโฟลเดอร์แทน
   - **หมวดหมู่** = จาก `[วงเล็บเหลี่ยม]` ในชื่อ หรือเดาจากคำ เช่น
     "ไหว้ครู" → วันสำคัญ, "ทัศนศึกษา" → ทัศนศึกษา, "ค่าย" → ค่าย
   - **ปีการศึกษา** = คำนวณจากวันที่ (พ.ค.–เม.ย. นับเป็นปีการศึกษาเดียวกัน)
4. ตรวจและแก้ไขข้อมูลในตารางได้ทุกช่อง ติ๊กออกรายการที่ไม่ต้องการ
5. กด **นำเข้าอัลบั้มที่เลือก** → ระบบนำเข้าทีละรายการพร้อมแสดงความคืบหน้า
   (สแกนรูปปกให้ทุกอัลบั้มอัตโนมัติ เปิดหน้าค้างไว้จนเสร็จ)

โฟลเดอร์ที่เคยนำเข้าแล้วจะขึ้นป้าย "มีแล้ว" และถูกข้ามอัตโนมัติ
ดังนั้น**สแกนโฟลเดอร์ใหญ่เดิมซ้ำได้เรื่อยๆ** เมื่อมีกิจกรรมใหม่เพิ่มเข้ามาระหว่างปี
ระบบจะนำเข้าเฉพาะโฟลเดอร์ใหม่เท่านั้น

### การตั้งชื่อโฟลเดอร์ให้ระบบอ่านแม่นที่สุด

| ชื่อโฟลเดอร์ | ผลที่ระบบอ่านได้ |
|---|---|
| `2569-06-12 กีฬาสี [กีฬา]` | ชื่อ "กีฬาสี", 12 มิ.ย. 2569, หมวดกีฬา — ครบอัตโนมัติ |
| `กีฬาสี 12-06-2569` | ชื่อ "กีฬาสี", วันที่ครบ, หมวดเดาจากคำว่า "กีฬา" |
| `วันไหว้ครู` | ชื่อครบ, หมวด "วันสำคัญ" (จากคำ), วันที่ = วันที่สร้างโฟลเดอร์ |
| `IMG_Backup_01` | ชื่อตามนั้น หมวด "ทั่วไป" — ควรแก้ในตารางก่อนนำเข้า |

รองรับปีทั้ง พ.ศ. (25xx) และ ค.ศ. (20xx) คั่นด้วย `-` `_` `.` หรือ `/` ก็ได้

> **เพิ่มคำเดาหมวดหมู่เอง:** แก้ตาราง `CATEGORY_KEYWORDS` ต้นส่วน Bulk Import
> ในไฟล์ `admin.js` เช่นเพิ่ม `'ธนาคารโรงเรียน': 'กิจกรรมพิเศษ'`

> หลังนำเข้าเสร็จ อย่าลืมกด **ดัชนีใบหน้า** ของแต่ละอัลบั้ม
> ถ้าต้องการให้อัลบั้มนั้นค้นหาด้วยใบหน้าได้

## การแก้ปัญหาที่พบบ่อย

| อาการ | สาเหตุ / วิธีแก้ |
|---|---|
| ขึ้น "เชื่อมต่อระบบไม่ได้" | ตรวจ `API_URL` ใน app.js/admin.js ว่าถูกต้องและลงท้ายด้วย `/exec`, ตรวจการ Deploy ว่าเป็น Execute as: Me + Anyone |
| รูปปกไม่แสดง | โฟลเดอร์ยังไม่แชร์แบบ "ทุกคนที่มีลิงก์" — ตั้งค่าแชร์แล้วกด "สแกนใหม่" |
| เพิ่มอัลบั้มแล้วขึ้น "ไม่พบรูปภาพ" | โฟลเดอร์ว่าง หรือรูปอยู่ในโฟลเดอร์ย่อย (ระบบสแกนเฉพาะชั้นแรก) |
| แก้ Code.gs แล้วไม่มีผล | ลืม Deploy เวอร์ชันใหม่ — ใช้ Manage deployments → Edit → New version |
| อัลบั้มไม่ขึ้นบนปฏิทิน | ไม่ได้กรอก "วันที่จัดกิจกรรม" ตอนเพิ่มอัลบั้ม — แก้ได้โดยลบแล้วเพิ่มใหม่ หรือแก้ในชีต Albums คอลัมน์ date (รูปแบบ yyyy-mm-dd) |
| login ไม่ผ่านทั้งที่รหัสถูก | ตรวจ Script Properties ว่า `ADMIN_USER`/`ADMIN_PASS` ตรงกับที่กรอก |
| กดค้นหาใบหน้าแล้วขึ้น "ยังไม่ได้สร้างดัชนี" | ครูยังไม่ได้กดปุ่ม "ดัชนีใบหน้า" ในแผงผู้ดูแลของอัลบั้มนั้น |
| กล้องไม่ขึ้น / ขอสิทธิ์ไม่สำเร็จ | ตรวจว่าเปิดผ่าน `https://` (ลิงก์ GitHub Pages จริง ไม่ใช่เปิดไฟล์ในเครื่อง) และเบราว์เซอร์ไม่ได้บล็อกกล้องของเว็บนี้ |
| สแกนดัชนีแล้วขึ้น "load failed" หลายรูป | โฟลเดอร์ยังไม่แชร์ "ทุกคนที่มีลิงก์" — ตั้งค่าแชร์แล้วกดสแกนดัชนีต่อ |
| ค้นหาแล้วไม่เจอตัวเองทั้งที่มีรูป | ลองถ่ายใหม่ในที่แสงดีขึ้น หน้าตรง ไม่ใส่หน้ากาก/แว่นดำ — หรือรูปในอัลบั้มเป็นภาพมุมไกลที่ใบหน้าเล็กเกินไป |
| รัน setup() แล้วไม่มีชีต FaceIndex | ใช้ Code.gs เวอร์ชันเก่า — วางโค้ดเวอร์ชันใหม่แล้วรัน setup() อีกครั้ง |

## การปรับแต่งเพิ่มเติม

- **สีประจำโรงเรียน:** แก้ตัวแปร `--navy` และ `--gold` ที่ต้นไฟล์ `style.css`
- **ชื่อโรงเรียน:** แก้ข้อความในส่วน `<header>` ของ `index.html`
- **จำนวนรูปปกที่แคช:** แก้ค่า `COVER_LIMIT` ใน `Code.gs` (ค่าเริ่มต้น 4)
- **ความเข้มงวดการจับคู่ใบหน้า:** แก้ `MATCH_THRESHOLD` ใน `app.js` (ค่าเริ่มต้น 0.6
  — ลดลงเช่น 0.55 จะเข้มงวดขึ้น ผลพลาดน้อยลงแต่บางรูปอาจหลุด, เพิ่มเป็น 0.65 จะเจอเยอะขึ้นแต่เสี่ยงจับผิดคน)
- ระบบนี้ต่อยอดได้อีก เช่น แจ้งเตือน Discord เมื่อมีอัลบั้มใหม่ หรือทำ PWA
  — โครงสร้าง API รองรับอยู่แล้ว
